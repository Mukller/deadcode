# deadcode

Статический анализатор Python-кода на основе AST. Находит неиспользуемые функции, классы, переменные и импорты — те, что только занимают место и вводят в заблуждение.

## Что находит

- Функции и методы, которые нигде не вызываются
- Классы без инстанциирования или наследования
- Переменные, которые присваиваются, но не читаются
- Импорты, которые нигде не используются
- Аргументы функций, которые всегда игнорируются

## Установка

```bash
pip install deadcode
# или
pipx install deadcode
```

## Использование

```bash
# Проверить весь проект
deadcode .

# Один файл
deadcode src/utils.py

# Исключить паттерны
deadcode . --ignore "test_*" --ignore "*_pb2.py"

# Только предупреждения (без ошибок)
deadcode . --warn-only

# Формат вывода
deadcode . --format json | jq .
```

## Пример вывода

```
src/utils.py:12: неиспользуемая функция 'parse_config'
src/models.py:45: неиспользуемый класс 'LegacyAdapter'
src/main.py:3: неиспользуемый импорт 'os'
src/handlers.py:78: неиспользуемая переменная 'tmp'

Итого: 4 проблемы в 4 файлах
```

## Что НЕ делает

Инструмент не запускает код и не выполняет динамический анализ. Это чистый AST-анализ. `getattr`, `__all__`, динамические вызовы через `importlib` — всё это вне его зоны ответственности.

Используй вместе с `mypy` и `ruff` для полного покрытия.
